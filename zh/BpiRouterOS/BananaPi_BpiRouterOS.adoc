= 系统介绍 

BpiRouter操作系统是由Banana Pi开源社区支持的一套路由器的现代化的Web管理界面，替代传统 LuCI，提供一套对标消费级精品路由的现代化图形化管理体验。

== 特性

* 底层系统：基于 OpenWrt v25.12.2（Linux 内核 6.12）开发；本系统是其上的 Web 管理界面 + 统一多板固件（A/B  OTA + 恢复）。
* 访问方式：浏览器打开 https://<路由器IP> （默认 https://192.168.100.1 ）。首次为自签名证书，浏览器会提示一次安全警告，确认继续即可。
* 界面语言：中文 / 英文，可切换。
* 显示模式：浅色 / 深色，跟随系统或手动切换。
* 响应式：桌面、平板、手机自适应（手机上侧边栏自动折叠）。
* 高级配置后路：界面保留 LuCI 入口，本界面未覆盖的高级/诊断场景可转 LuCI。

== 支持设备

* 支持设备：6 款 BPI 路由板—R4 / BPI-R4Lite / BPI-R4 Pro / BPI-R3 / BPI-R3 Mini / OpenWrt One。Web 界面在所有板上完全一致。

= 功能说明

== 首次开机配置向导

作用：路由器全新（未设管理密码）时，引导完成最基本的上网与安全设置。

操作：首次打开管理页会自动进入「设置向导」，分 3 步：

* 1，上网方式（WAN）：选择 DHCP（自动获取）/ 静态 IP / PPPoE（宽带拨号，填账号密码）。
* 2，无线（WiFi）：设置 WiFi 名称（SSID）与密码。
* 3,管理员密码：设置管理后台的登录密码

预期效果（可验证点）：

* 未设密码前，访问任何页面都被引导到向导。
* 完成向导后自动登录进入仪表盘，无需再单独登录一次。
* 向导只在「未设密码」时可用；一旦设过密码，向导入口关闭，再访问直接进登录页。

== 登录与用户角色

登录操作：输入用户名 + 密码登录（默认管理员用户名 root ）。

预期效果（可验证点）：

* 密码正确 → 进入仪表盘；密码错误 → 提示「凭据无效」。
* 防爆破：同一来源连续 5 次失败后，登录开始逐次延时（1/2/4/8/16/32 秒，最长 60秒），界面提示「失败次数过多，请 N 秒后重试」；一次成功登录立即解除。
* 登录状态有效期 1 小时，过期后自动跳回登录页。

* 即使绕过界面直接调用，越权操作也会被后端拒绝（返回「权限不足」）。
* 「固件升级 / 备份恢复 / 用户管理 / 审计日志 / DMZ」等卡片仅管理员可见。

== 仪表盘（首页）

作用：一屏总览路由器运行状态，约每 5 秒自动刷新。包含信息：

* 资源卡片：CPU 占用率与温度、内存使用、存储占用。
* WAN 状态：联网状态、获取到的 IP。
* 客户端数：当前连接设备数 / 在线数。
* 实时流量曲线：WAN 口下载 / 上传速率的滚动折线图。网络接口列表：各接口（含 IPv4 / IPv6 双栈）的状态。
* 无线列表：各频段的信道与连接数。

预期效果（可验证点）：

* 数据随时间自动刷新（无需手动刷新页面）。
* 流量曲线随实际上下行变化起伏。
* 接口同时展示 IPv4 与 IPv6（如 wan / wan6）。

== 有线网络

=== WAN（上网方式）

作用：配置路由器如何接入互联网。

操作：选择 DHCP / 静态 IP / PPPoE 之一并填写对应参数（静态需 IP/掩码/网关/DNS；PPPoE 需账号密码），保存。

预期效果（可验证点）：

* 切换上网方式后，不适用的旧字段被自动清除（如从 PPPoE 切到 DHCP 不会残留旧账号）。
* 静态模式下 DNS 按当前表单整体生效。
* 保存为「连通性变更」，受应用确认保护（见 §11.1）——若改错导致掉线，会自动回滚。

=== LAN（局域网）

作用：配置路由器自身的局域网 IP 与 DHCP 地址分配。

操作：设置 LAN IP / 子网掩码、DHCP 开关、地址池起始 / 数量 / 租期，保存。

预期效果（可验证点）

* 子网掩码必须是合法的点分十进制掩码，非法值被拒绝。
* LAN IP 与 DHCP 设置一起生效，不会出现「IP 改了但地址池没改」的错配。
* 改 LAN IP 同样受应用确认保护。
