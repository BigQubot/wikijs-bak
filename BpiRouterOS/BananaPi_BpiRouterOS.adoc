=  1. What is this

**BpiRouter OS** It is a unified firmware designed for the Banana Pi router series - built on top of OpenWrt，provide one**Modern graphical WEB management interface**，And a set of **firmware upgrade and recovery mechanisms to prevent bricks**。

It needs to solve two pain points：

- **Bare OpenWrt / LuCI is not user-friendly for ordinary users.** → We offer **a graphical interface benchmarked against consumer-grade high-quality routers, an out-of-the-box configuration wizard, and quick access to common functions on a single screen.**

- **It's easy to damage a router when flashing its firmware, and it's difficult to rescue a bricked router.** → We provide dual slots for firmware A/B, automatic fallback in case of online upgrade failure, and a four - level recovery ladder, so the device can be rescued even if the firmware flashing goes wrong.

**One set of firmware is compatible with 6 types of boards **：The same Web interface and functions run on all six BPI router boards. The differences between the boards are only at the hardware level, and the firmware automatically adapts to each board.

=  4.Function overview

==  4.1 Out of the box
- **First configuration wizard**：When a brand-new device is powered on, it will automatically enter the wizard. You can set up the Internet connection mode (DHCP / Static / Broadband Dial-up), Wi-Fi, and the management password in 3 steps.

- **Dashboard**：One-screen overview of CPU, temperature, memory, storage, network connection status, and number of clients、**display the real-time upstream and downstream traffic curve**，automatic refresh。

==  4.2 Network and Wireless

- **Internet（WAN）**：DHCP / Static IP / PPPoE broadband dial - up service，IPv4 + IPv6 support。
- **Local Area Network（LAN）**：LAN IP、DHCP Address allocation (Address pool / Lease term)。
- **Wireless**：Multi - band management（2.4G / 5G / 6G），Channel/Bandwidth/Power/Country code，SSID and Encryption（6GHz Compulsory WPA3/OWE）。
- **Connected device**：View the connected devices in real-time IP / MAC / Hostname / Online status。

==  4.3 Surf the Internet safely
- **VPN**：built - in **WireGuard** and **OpenVPN**——two protocols**can be enabled simultaneously**，It can serve as both a client (connecting to a remote VPN) and a server (making the local machine a VPN server). The server allows users to customize the listening port and address range，**servergenerate configuration with one click + QR code**（Mobile App Scan the code and connect immediately），And support viewing and revoking the client list.
- **Port forwarding **：Map the external network ports to the internal network devices.
- **DMZ**：Forward all inbound external network traffic to the specified internal network host.
- **Firewall rules **：Customize allow / deny / discard rules.
- **DDNS**：dynamic public IP binding to a fixed domain name.（Support DuckDNS / No-IP / Dynu / Cloudflare / FreeDNS）。

==  4.4 System management
- **Hostname / Time zone / System time **：Select the time zone from the drop-down list (searchable). The time is automatically synchronized via NTP. Manual setting is also supported, as well as synchronization with the browser time (used as a fallback for time calibration in scenarios without RTC or offline).
- **Change password,Reset**。
- **Secondary confirmation of key changes**：newwork / WiFi / firewalla...Changes that may cut off the management connection，You can enable "Time-limited confirmation after saving" - when a disconnection occurs due to error correction, it will automatically roll back. It is enabled by default.
- **Firmware upgrade**(Dual slots A/B)、**Configuration backup and restoration**、**Restore to settings factory**。
- **Diagnostic tool**：Ping / Traceroute / DNS inquire，The results can be directly viewed on the WEB UI。
- **System log**。

==  4.5 Multi - user and auditing
- **Three types of roles**：Administrator (full access) / Operator (daily configuration) / Observer (read-only). Minimize permissions. Observers cannot see or modify the configuration.
- **Audit log**（Only administrator）：Record security-related operations such as logins, permission denials, configuration changes, firmware upgrades, and user management for post-event traceability (no passwords/keys are recorded).

=  5. Reliability

**Upgrade and recovery without the risk of bricking forever**.

This is the most core differentiating capability of BpiRouter OS. Router firmware most worried "bricking during upgrade, becoming remotely inoperable, and having to be sent back for repair" —— we use dual slots + four - layer backup to completely eliminate this risk.

==  5.1 firmware A/B dual - slot upgrade

The device has **two firmware slots**. During online upgrade,：

1. Write the new firmware to the **backup slot** without touching the currently running slot.；

2. Automatically restart and enter the new firmware. A **"Upgrade to be confirmed"** banner with a countdown will pop up on the interface.

3. Confirm that everything is normal. → check「**confirm upgrade**」become a full ；

4. **If not confirmed (for example, the network is not working with the new firmware or the person is not present)→ Automatically roll back when the countdown ends.**revert to the firmware and configuration before the upgrade.

Before upgrading, the firmware will be automatically verified, and incompatible images will be directly rejected. You can choose to **"retain configuration"** to keep the settings consistent after the upgrade.


==  5.2 Four - layer recovery ladder

**From soft to hard, provide support at each level:**

[options="header",cols="1,2,3"]
|====
| Level | Trigger mode | Effect
| **L1 Configuration rollback** | Automatic | Changing the configuration that causes disconnection → Automatic cancellation if not confirmed within the time limit

| **L2 Automatic firmware rollback** | Automatic | Failed to start the bad firmware → The boot automatically switches back to the previous good firmware slot.

| **L3 Restore to factory settings** | During operation, **press and hold the restore button for about 5 seconds**. | Clear the current configuration and restore to factory settings (without reflashing the firmware)
| **L4 Full device rescue from brick** | **Power off → Hold the restore button → Power on** and keep holding | Flash the whole device back to the factory settings from the built-in gold image.
|====

> **Result**：Even if both the configuration and the firmware are damaged, L4 can still restore the device to its factory settings.——**Even if the device is bricked during the flashing process, it can still be rescued and won't become a permanent brick**。
>
> 💡 **The recovery button varies by board**：BPI-R4 use **WPS key**；R4-Lite / R4 Pro / R3 / R3 Mini use **physical reset button**.Please refer to the instructions for each board.
