# BpiRouter OS —  burning guide

> Burn a board to make it bootable with the factory system. This page serves as an operation guide for users; for the complete technical details (bit-by-bit DIP switch settings, light state timing, and step-by-step serial port procedures) on the production line/for maintenance, refer to the factory burn instructions that come with the firmware.

[← link:/en/BpiRouterOS/BananaPi_BpiRouterOS[Return home page]]

---

## 0. Must-read before starting

- ⚠️ **Diping-switch (boot media selection) is the step most prone to errors and the most with serious consequences.** Dialing the wrong switch will only result in booting to the wrong media and will not damage the hardware, but it will make the process [seem unresponsive].**The complete definition of dip-switching shall be based on the silkscreen on the actual board and the schematic diagram.。**
- **Power off first and then switch the DIP switches / plug and unplug** throughout the process, avoid hot plugging.
- **Do not cut off the power during the programming and writing process.** The system has CRC verification for writing (a bad image will never be written), but a power outage during the process may still result in incomplete writing.
- Pay attention to anti-static (ESD). Some boards have requirements for power supply.（For example, the BE14 triple-band of R4-Lite requires 12V）。

---

## 1.Select the programming method 

Depending on whether your board has an SD card slot, choose one of the two:

[options="header",cols="2,3,5"]
|====
| Method | applicable board | When to use 
| **Method A — SD card** | R4 / R4-Lite / R4 Pro / R3（**with SD slot**） | Main process，Preferred
| **Method B — Serial port + USB flash drive** | R3 Mini / OpenWrt One（**without SD slot**）；And the brick machine rescue for any board. | A board an without SD slot, or a completely bricked board that needs to be recovered from the bottom layer.
|====

> Why are there two types? For boards with an SD slot, the SD card comes with a bootable installer. When the card is inserted and the power is turned on, it will automatically flash without the need for a serial port. For boards without an SD slot, the first boot program can only be sent into the memory via the serial port through the chip's BootROM.

---

## 2. Prepare image

Get the files corresponding to your board type from the **download page**:

- **Method A**：`*-sdcard.img`（Installation image for written to the SD card）。
- **Method B**：Comes with the firmware**flashkit** toolkit，including memory bootstrap progra、NAND Golden mirror `snand-factory.bin` and verification file `snand-factory.crc`、Serial port download tool and step-by-step instructions。

> ⚠️ **`snand-factory.bin` 与 `snand-factory.crc` 必须来自同一次构建、一起拷贝。** 烧录前会用 .crc 校验 .bin，不匹配则绝不写入——这是防坏镜像保护，但也意味着两文件版本错配会「永远校验失败、写不进」。

---

## 3. 方式 A：SD 卡烧录（三跳）

有 SD 槽的板产品最终运行在 eMMC；由于 SD 与 eMMC 复用同一控制器，需借 NAND 中转。流程是 **SD 启动写 NAND → NAND 启动写 eMMC → eMMC 启动进系统**，每跳都不进操作系统，看到完成提示即断电换拨码。

1. **制卡**：把 `*-sdcard.img` 写入 SD 卡（用 Etcher / `dd` 等）。
2. **第 1 跳**：拨到 **SD 启动** → 插卡上电 → 写入 NAND，完成后断电。
3. **第 2 跳**：拨到 **NAND 启动** → 上电 → 自动装入 eMMC，完成后断电。
4. **第 3 跳**：拨到 **eMMC 启动** → 上电 → 进入系统（A 槽）。SD 卡可拔出。

> 💡 **关键提醒：每一跳是「从 A 启动、写入 B」，A 和 B 的拨码都要对。** 第 1 跳除了拨 SD 启动，**NAND 对应的跳线也要选对**；第 2 跳除了拨 NAND 启动，**eMMC 对应的 pin 也要选对**。否则会「看似没反应」。
>
> 💡 **完成判据以串口输出为准**；接一路串口（115200 8N1）能看到每跳的写入进度与完成提示。部分板另有 LED 辅助（写入时点亮、完成熄灭）。

各板的拨码关键位、灯态、特别注意（如 R4-Lite 的 NOR/NAND 开关、BE14 需 12V）详见随固件附带的工厂烧录说明。

---

## 4. 方式 B：串口 + U 盘烧录

适用 R3 Mini / OpenWrt One，以及任意板的砖机救援。大致流程：

1. 按 flashkit 说明，用串口下载工具把内存引导程序送进设备内存（不写 flash）。
2. 把 `snand-factory.bin` + `snand-factory.crc` 拷到 **FAT32 U 盘**根目录，插入设备 USB 口。
3. 设备**自动检测 U 盘并写入 NAND**（写前 CRC 校验、写后回读校验）。
4. R3 Mini 还需一步：拔掉 U 盘、拨到 NAND 启动，自动把系统装入 eMMC；OpenWrt One 是纯 NAND，一步写完即可。

逐步命令、接线（UART0 / Type-C）、灯态见固件附带的串口烧录说明（flashkit 工具包内含对应板型的逐步指南）。

---

## 5. 烧录后验证

进入系统后：

1. 浏览器打开 `https://192.168.100.1`，应出现配置向导 → 继续到 [link:/zh/BpiRouterOS/BananaPi_BpiRouterOS_Setup[首次配置]]。
2. 如接了串口，可确认两个固件槽都能正常引导（A/B 双槽就绪）。

完成后即可正常使用。若中途出问题，见 [link:/zh/BpiRouterOS/BananaPi_BpiRouterOS_Recovery[恢复与救砖]]。
