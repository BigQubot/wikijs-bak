# BpiRouter OS —  burning guide

> Burn a board to make it bootable with the factory system. This page serves as an operation guide for users; for the complete technical details (bit-by-bit DIP switch settings, light state timing, and step-by-step serial port procedures) on the production line/for maintenance, refer to the factory burn instructions that come with the firmware.

---

## 0. Must-read before starting

- ⚠️ **Diping-switch (boot media selection) is the step most prone to errors and the most with serious consequences.** Dialing the wrong switch will only result in booting to the wrong media and will not damage the hardware, but it will make the process [seem unresponsive].**The complete definition of dip-switching shall be based on the silkscreen on the actual board and the schematic diagram.。**
- **Power off first and then switch the DIP switches / plug and unplug** throughout the process, avoid hot plugging.
- **Do not cut off the power during the programming and writing process.** The system has CRC verification for writing (a bad image will never be written), but a power outage during the process may still result in incomplete writing.
- Pay attention to anti-static (ESD). Some boards have requirements for power supply.（For example, the BE14 triple-band of R4-Lite requires 12V）。

## 1.Select the programming method 

Depending on whether your board has an SD card slot, choose one of the two:

[options="header",cols="2,3,5"]
|====
| Method | applicable board | When to use 
| **Method A — SD card** | R4 / R4-Lite / R4 Pro / R3（**with SD slot**） | Main process，Preferred
| **Method B — Serial port + USB flash drive** | R3 Mini / OpenWrt One（**without SD slot**）；And the brick machine rescue for any board. | A board an without SD slot, or a completely bricked board that needs to be recovered from the bottom layer.
|====

> Why are there two types? For boards with an SD slot, the SD card comes with a bootable installer. When the card is inserted and the power is turned on, it will automatically flash without the need for a serial port. For boards without an SD slot, the first boot program can only be sent into the memory via the serial port through the chip's BootROM.

---

## 2. Prepare image

Get the files corresponding to your board type from the **download page**:

- **Method A**：`*-sdcard.img`（Installation image for written to the SD card）。
- **Method B**：Comes with the firmware**flashkit** toolkit，including memory bootstrap progra、NAND Golden mirror `snand-factory.bin` and verification file `snand-factory.crc`、Serial port download tool and step-by-step instructions。

> ⚠️ **`snand-factory.bin` and `snand-factory.crc`  must come from the same build and be copied together.**Before programming, the .bin file will be verified using the .crc file. If they don't match, writing will never be performed. This is a protection mechanism against bad images，But it also means that if the versions of the two files are mismatched, it will「verification always fails and writing is not possible.」。

---

## 3. Method A：SD burn（Triple jump）

Board products with SD slots will ultimately run on. eMMC；Since SD and eMMC share the same controller, NAND needs to be used for transit.The process is**SD boot to write NAND → NAND boot to write eMMC → eMMC boot into the system **，Don't enter the operating system at each jump. When you see the completion prompt, power off and change the DIP switch.

1. **make SD card**：Write *-sdcard.img to the SD card.（use Etcher / `dd` ）。
2. **1 jump **：Dial to **SD boot** → Insert the card and power on → Write to NAND, then power off after completion.
3. **2 jump**：Dial to **NAND boot** → Power on → Automatically load eMMC, then power off after completion.
4. **3 jump**：Dial to **eMMC boot** → Power on → Enter the system (Slot A). The SD card can be removed.

> 💡 **Key reminder: Each hop is [start from A and write to B]. The DIP switches of both A and B must be set correctly.** In the first jump, apart from booting by inserting the SD card,，**The corresponding jumper for NAND also needs to be selected correctly.**；In the second jump, in addition to setting it for NAND boot,，**The corresponding pins for eMMC also need to be selected correctly.**。Otherwise, it will [seem to have no response].。
>
> 💡 **The completion criterion is based on the serial port output.**；Connect to a serial port (115200 8N1), and you can see the writing progress and completion prompt for each jump. Some boards are additionally assisted by LEDs (the LEDs light up during writing and turn off upon completion).

For the key DIP switch positions, lamp states, and special notes (such as the NOR/NAND switch on the R4-Lite and the requirement of 12V for the BE14) of each board, please refer to the factory programming instructions attached to the firmware.

## 4. Method B：Serial port + USB flash drive burning

Suitable R3 Mini / OpenWrt One，And the rescue of brick machines for any board. General process:

1. According to the instructions of FlashKit, use the serial port download tool to send the memory boot program into the device's memory (without writing to the flash).
2. Copy `snand-factory.bin` + `snand-factory.crc` to **FAT32 USB flash drive**root directory and insert the device into the USB port.
3. Board **Automatically detect the USB flash drive and write to the NAND.**（CRC check before writing、Read-back check after writing）。
4. R3 Mini one more step is needed: unplug the USB flash drive and switch to NAND boot，Automatically install the system into the eMMC.；OpenWrt One is only NAND，It can be written in one step.

Step-by-step commands、wiring（UART0 / Type-C）、lamp status can be found in the serial port burning instructions attached to the firmware （flashkit toolkit contains step-by-step guides for the corresponding board types.）。

## 5. Verification after burning

进入系统后：

1. 浏览器打开 `https://192.168.100.1`，应出现配置向导 → 继续到 [link:/zh/BpiRouterOS/BananaPi_BpiRouterOS_Setup[首次配置]]。
2. 如接了串口，可确认两个固件槽都能正常引导（A/B 双槽就绪）。

完成后即可正常使用。若中途出问题，见 [link:/zh/BpiRouterOS/BananaPi_BpiRouterOS_Recovery[恢复与救砖]]。
