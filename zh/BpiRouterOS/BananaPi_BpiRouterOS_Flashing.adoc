# BpiRouter OS — 烧录指南

> 把一块板烧录成可启动的出厂系统。本页是用户向操作向导；产线/维修的完整技术细节（逐位拨码、灯态时序、串口逐步流程）见随固件附带的工厂烧录说明。
---

## 0. 开始前必读

- ⚠️ **拨码（启动介质选择）是最易出错、后果最重的一步。** 拨错只会启动到错介质、不会损坏硬件，但会让流程「看起来没反应」。**完整拨码定义以板实物丝印 + 原理图为准。**
- 全程 **先断电再拨码 / 插拔**，避免热插拔。
- **烧录写入期间不要断电。** 系统对写入有 CRC 校验（坏镜像绝不写入），但中途断电仍可能写半截。
- 注意防静电（ESD）。部分板对供电有要求（如 R4-Lite 的 BE14 三频需 12V）。

---

## 1. 选择烧录方式

按你的板有没有 SD 卡槽，二选一：

**| 方式 | 适用板 | 何时用 |**

|---|---|---|

| **方式 A — SD 卡** | R4 / R4-Lite / R4 Pro / R3（**有 SD 卡槽**） | 主流程，首选 |

| **方式 B — 串口 + U 盘** | R3 Mini / OpenWrt One（**无 SD 卡槽**）；及任意板的砖机救援 | 无 SD 槽的板，或彻底变砖需从底层救回 |

> 为什么分两种：有 SD 槽的板，SD 卡自带可启动的安装器，插卡上电即自动烧录、无需串口；无 SD 槽的板只能靠串口经芯片 BootROM 把第一份引导程序送进内存。

---

## 2. 准备文件

从 **下载页** 获取你的板型对应文件：

- **方式 A**：`*-sdcard.img`（写入 SD 卡的安装镜像）。
- **方式 B**：固件附带的 **flashkit** 工具包，含内存引导程序、NAND 黄金镜像 `snand-factory.bin` 及其校验文件 `snand-factory.crc`、串口下载工具与逐步说明。

> ⚠️ **`snand-factory.bin` 与 `snand-factory.crc` 必须来自同一次构建、一起拷贝。** 烧录前会用 .crc 校验 .bin，不匹配则绝不写入——这是防坏镜像保护，但也意味着两文件版本错配会「永远校验失败、写不进」。

---

## 3. 方式 A：SD 卡烧录（三跳）

有 SD 槽的板产品最终运行在 eMMC；由于 SD 与 eMMC 复用同一控制器，需借 NAND 中转。流程是 **SD 启动写 NAND → NAND 启动写 eMMC → eMMC 启动进系统**，每跳都不进操作系统，看到完成提示即断电换拨码。

1. **制卡**：把 `*-sdcard.img` 写入 SD 卡（用 Etcher / `dd` 等）。
2. **第 1 跳**：拨到 **SD 启动** → 插卡上电 → 写入 NAND，完成后断电。
3. **第 2 跳**：拨到 **NAND 启动** → 上电 → 自动装入 eMMC，完成后断电。
4. **第 3 跳**：拨到 **eMMC 启动** → 上电 → 进入系统（A 槽）。SD 卡可拔出。

> 💡 **关键提醒：每一跳是「从 A 启动、写入 B」，A 和 B 的拨码都要对。** 第 1 跳除了拨 SD 启动，**NAND 对应的跳线也要选对**；第 2 跳除了拨 NAND 启动，**eMMC 对应的 pin 也要选对**。否则会「看似没反应」。
>
> 💡 **完成判据以串口输出为准**；接一路串口（115200 8N1）能看到每跳的写入进度与完成提示。部分板另有 LED 辅助（写入时点亮、完成熄灭）。

各板的拨码关键位、灯态、特别注意（如 R4-Lite 的 NOR/NAND 开关、BE14 需 12V）详见随固件附带的工厂烧录说明。

---

## 4. 方式 B：串口 + U 盘烧录

适用 R3 Mini / OpenWrt One，以及任意板的砖机救援。大致流程：

1. 按 flashkit 说明，用串口下载工具把内存引导程序送进设备内存（不写 flash）。
2. 把 `snand-factory.bin` + `snand-factory.crc` 拷到 **FAT32 U 盘**根目录，插入设备 USB 口。
3. 设备**自动检测 U 盘并写入 NAND**（写前 CRC 校验、写后回读校验）。
4. R3 Mini 还需一步：拔掉 U 盘、拨到 NAND 启动，自动把系统装入 eMMC；OpenWrt One 是纯 NAND，一步写完即可。

逐步命令、接线（UART0 / Type-C）、灯态见固件附带的串口烧录说明（flashkit 工具包内含对应板型的逐步指南）。

---

## 5. 烧录后验证

进入系统后：

1. 浏览器打开 `https://192.168.100.1`，应出现配置向导 → 继续到 [首次配置](./setup.md)。
2. 如接了串口，可确认两个固件槽都能正常引导（A/B 双槽就绪）。

完成后即可正常使用。若中途出问题，见 [恢复与救砖]。
